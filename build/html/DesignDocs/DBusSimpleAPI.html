Simple D-Bus API design¶

Related ticket(s):

  • ​Create a library to simplify usage of D-Bus responder

Problem Statement¶

Using D-Bus requires a significant amount of knowledge of D-Bus and its underlying library API. Libraries like libdbus or libdbus_glib are quite complex and requires a lot of code to do even the simplest things. The purpose of this document is to describe a new public API to access most fundamental parts of SSSD’s D-Bus repsonder in a simple way so that a user doesn’t have to deal with D-Bus at all.

Prerequisites¶

  • Each attribute of every D-Bus object accessible via this API is represented as string.
  • Naming convention of D-Bus methods:
    • List<class><condition>(arg1, ...) returning array of object paths
      • ListUsers()
      • ListDomains()
      • ListUsersByName(filter)
      • ListGroupsByName(filter)
    • Find<class><condition>(arg1, ...) returning single object path
      • FindUserById(id)
      • FindDomainByName(name)
