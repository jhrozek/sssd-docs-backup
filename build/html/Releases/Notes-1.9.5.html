Highlights¶

  • This release focused mainly on fixing regressions compared to the 1.8 series and bugfixes for features introduced in the 1.9 release cycle. The release also includes one security fix
  • Includes a fix for ​CVE-2013-0287: A simple access provider flaw prevents intended ACL use when SSSD is configured as an Active Directory client
  • Fixed spurious password expiration warning that was printed on login with the Kerberos back end
  • A new option ldap_rfc2307_fallback_to_local_users was added. If this option is set to true, SSSD is be able to resolve local group members of LDAP groups.
  • Fixed an indexing bug that prevented the contents of autofs maps from being returned to the automounter deamon in case the map contained a large number of entries
  • Several fixes for safer handling of Kerberos credential caches for cases where the ccache is set to be stored in a DIR: type
