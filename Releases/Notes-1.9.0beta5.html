Highlights

  • Many fixes for the support for setting default SELinux user context from FreeIPA, most notably fixed the specificity evaluation
  • Fixed an incorrect default in the krb5_canonicalize option of the AD provider which was preventing password change operation
  • The shadowLastChange attribute value is now correctly updated with the number of days since the Epoch, not seconds
